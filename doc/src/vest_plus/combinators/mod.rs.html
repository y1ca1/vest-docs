1//! Combinators for composing binary data formats.
2//!
3//! # Primitive combinators
4//!
5//! | Combinator | Description |
6//! |---|---|
7//! | [`Fixed<N>`] | Exactly `N` bytes |
8//! | [`Varied<Len>`] | Variable-length bytes determined by a length parameter |
9//! | [`U8`] | Unsigned 8-bit integer |
10//! | [`U16Le`] / [`U16Be`] | Unsigned 16-bit integer (little/big-endian) |
11//! | [`U32Le`] / [`U32Be`] | Unsigned 32-bit integer (little/big-endian) |
12//!
13//! # Higher-order combinators
14//!
15//! | Combinator | Description |
16//! |---|---|
17//! | [`(A, B)`](tuple::spec) | Sequential composition |
18//! | [`Choice<A, B>`] | Non-malleable ordered alternative |
19//! | [`Alt<A, B>`] | Malleable ordered alternative |
20//! | [`Opt<A>`] | Optional value |
21//! | [`Optional<A, B>`] | Same as `(Opt<A>, B)`, but disambiguates `A` and `B` |
22//! | [`Star<A>`] | The Kleene star: zero-or-more repetitions |
23//! | [`Repeat<A, B>`] | Same as `(Star<A>, B)`, but disambiguates `A` and `B` |
24//! | [`RepeatN<C, Len>`] | Fixed number of repetitions determined by a length parameter |
25//! | [`Array<N, C>`] | Array of values of length `N` |
26//! | [`Preceded<A, B>`] | Same as `(A, B)`, but discards A's value |
27//! | [`Terminated<A, B>`] | Same as `(A, B)`, but discards B's value |
28//! | [`Mapped<Inner, M>`] | Isomorphic format transformation via a [bijection](mapped::spec::Mapper) |
29//! | [`Refined<Inner, Pred>`] | Format refinement via a [predicate](crate::core::spec::SpecPred) |
30//! | [`Tag<Inner, T>`] | Matches a specific constant value |
31//! | [`Tagged<T, Of>`] | Same as `Preceded<Tag<T, T::Val>, Of>` (for convenience) |
32//! | [`Cond<Inner>`] | Boolean-gated combinator (most often used in branches of `Choice` / `Alt`) |
33//!
34//! # Dependent combinators
35//!
36//! | Combinator | Description |
37//! |---|---|
38//! | [`Implicit<A, B>`] | Like `Preceded<A, B>`, but `B` can depend on `A`'s value |
39//! | [`Bind<A, B>`] | Like `Implicit<A, B>`, but `B` must be one of the [dependent family of combinators](DepCombinator) |
40//!
41//! ## Dependent family of combinators
42//!
43//! Each combinator in this family explicitly declares the value(s) it depends on during parsing, and provides a recovery function
44//! to reconstruct those values during serialization (see [`DepCombinator`] for details).
45//!
46//! | Combinator | Dependency | Description |
47//! |---|---|---|
48//! | [`dependent::VLData<Len>`] | Any value that can be used [as a length parameter](AsLen) | Length-prefixed raw bytes |
49//! | [`dependent::VLDataOf<Len, C>`] | Same as `VLData` | Length-prefixed bytes interpreted by format `C` |
50//! | [`dependent::TVOr<Tag, C, Rest>`] | Any value that can be used as a tag | A chain of tagged unions |
51//! | [`dependent::Uninhabited<Tag>`] | Any value | End of a tagged union chain |
52//! | [`dependent::TLVOf<Tag, Len, Body>`] | A pair of `(Tag, Len)` | A TLV (tag-length-value) format |
53//!
54//! # Tail combinators
55//!
56//! | Combinator | Description |
57//! |---|---|
58//! | [`Tail`] | Like [`Varied`], but at the tail position (underspecify the format and allow trailing data) |
59//! | [`Eof`] | Signals end-of-file (no trailing data) |
60//! | [`OptionalEof<C>`] | Same as `Optional<C, Eof>` (for convenience) |
61//! | [`RepeatUtilEof<C>`] | Same as `Repeat<C, Eof>` (for convenience) |
62//!
63//! # Marker combinators
64//!
65//! | Combinator | Description |
66//! |---|---|
67//! | [`Empty`] | Unit (nothing interesting, but still occupies zero bytes) |
68//! | [`Void`] | Bottom (no value can satisfy this format) |
69//!
70//! # Recursive combinators
71//!
72//! | Combinator | Description |
73//! |---|---|
74//! | [`Fix<LIMIT, Body>`] | (Compile-time) bounded fixpoint for recursive formats |
